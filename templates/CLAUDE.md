## 1. 코딩 전 먼저 생각하라 (Think Before Coding)

- **가정하지 말 것**: 불확실한 부분은 숨기지 말고 표면으로 드러낼 것.
- **해석이 여러 가지라면**: 각각을 명시적으로 제시하고 어느 쪽인지 확인할 것.
- **혼란스러우면 물어볼 것**: 잘못된 가정으로 구현한 뒤 되돌리는 것보다 먼저 묻는 게 낫다.
- **더 나은 방법이 있다면 제안**: 요청이 최선이 아닐 때는 트레이드오프를 설명하고 대안을 제시할 것.

## 2. 단순함 우선 (Simplicity First)

- **문제를 해결하는 최소한의 코드**만 작성할 것.
- 금지: 요청하지 않은 기능 / 미래를 위한 추상화 / 발생 불가능한 에러 핸들링 / 불필요한 유연성.
- 자가 점검: "시니어 엔지니어가 이걸 보고 과하다고 할까?" → 그렇다면 줄일 것.

## 3. 외과적 변경 (Surgical Changes)

- **반드시 바꿔야 할 것만** 건드릴 것.
- 내가 만든 mess만 정리. 주변 코드 스타일 개선, 고장나지 않은 코드 리팩토링 금지.
- 변경된 모든 줄은 사용자 요청으로 직접 추적 가능해야 함.

## 4. 목표 주도 실행 (Goal-Driven Execution)

- 추상적인 태스크를 **검증 가능한 체크포인트**로 변환할 것.
- 성공 기준을 먼저 정의할 것. 검증 방법 없이 구현하지 말 것.
